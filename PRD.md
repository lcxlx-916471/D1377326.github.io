# 產品需求文件 (PRD)：任務管理系統

## 1. 產品概述 (Product Overview)
**系統描述**：這是一個專為個人用戶設計的任務管理系統，旨在提供一個簡單直觀的介面，讓使用者可以輕鬆地新增、追蹤、完成和刪除待辦事項。
**目標用戶**：需要管理日常工作、學習計畫或生活瑣事的個人用戶。

## 2. 產品目標 (Product Goals)
- **提升效率**：幫助用戶清晰掌握待辦事項，減少遺忘。
- **簡單易用**：提供低學習成本的操作介面，讓用戶能快速上手，無需繁雜的設定。
- **狀態追蹤**：讓用戶能隨時過濾並查看未完成與已完成的任務，獲得成就感並專注於當下目標。

## 3. 功能需求 (Functional Requirements)

### 3.1 核心功能
1. **新增任務 (Add Task)**
   - 用戶可以在輸入框中輸入任務名稱。
   - 支援按下「Enter」鍵或點擊「新增」按鈕來建立新任務。
   - 驗證：任務名稱不得為空字串，且可能有合理的字數限制（如 100 字內）。
2. **顯示所有任務清單 (View Task List)**
   - 系統預設顯示所有已新增的任務列表。
   - 列表應包含任務名稱與當前的完成狀態。
   - 當清單為空時，應顯示友善的提示文字（如：「目前尚無任務，趕快新增一個吧！」）。
3. **標記任務完成 (Mark Task as Completed)**
   - 每個任務項目旁提供一個核取方塊 (Checkbox) 或完成按鈕。
   - 點擊後，任務狀態切換為「已完成」，並在視覺上給予明顯反饋（例如：加上刪除線、文字顏色變淡、背景色改變）。
   - 支援取消完成狀態，恢復為「未完成」狀態。
4. **刪除任務 (Delete Task)**
   - 每個任務項目旁提供一個「刪除」按鈕（可使用垃圾桶圖示）。
   - 點擊後，將該任務從清單中永久移除。
5. **依完成狀態篩選 (Filter Tasks)**
   - 提供篩選控制項（如頁籤 Tab 或下拉選單）：
     - **全部 (All)**：顯示所有任務。
     - **進行中 (Active)**：僅顯示未完成的任務。
     - **已完成 (Completed)**：僅顯示已完成的任務。

## 4. 非功能需求 (Non-Functional Requirements)
- **使用者體驗 (UX)**：操作應有即時的反饋（如按鈕點擊動畫、刪除時的過場動畫），介面設計需簡潔美觀。
- **響應式設計 (Responsive Design)**：介面需支援桌面端 (Desktop) 與行動端 (Mobile) 瀏覽器，確保在不同螢幕尺寸下皆能正常使用。
- **資料儲存 (Data Storage)**：初版建議使用瀏覽器的 LocalStorage 儲存資料，確保使用者重整頁面或重新開啟瀏覽器後，任務資料不會消失。

## 5. 使用者流程 (User Flow)
1. **進入系統**：用戶打開網頁應用程式，載入並顯示先前的任務清單。
2. **新增任務**：在頂部的輸入框輸入任務內容（如：「寫報告」），按下 Enter 鍵，清單下方立刻新增該項目。
3. **完成任務**：報告寫完後，點擊該任務旁的核取方塊，任務被加上刪除線，狀態更新為已完成。
4. **篩選查看**：點擊「進行中」頁籤，專注於尚未完成的其他任務；或點擊「已完成」頁籤，回顧已完成的事項。
5. **清理任務**：對於不再需要執行的任務，點擊其旁邊的刪除按鈕將其移除。

## 6. 未來擴展建議 (Future Considerations)
這部分可作為後續版本 (v2.0+) 的迭代方向：
- 任務編輯功能（修改錯字或調整內容）。
- 任務截止日期與時間提醒。
- 任務優先度標籤（高、中、低）。
- 拖曳排序 (Drag and Drop) 調整任務優先順序。
- 註冊/登入系統與雲端資料同步。
